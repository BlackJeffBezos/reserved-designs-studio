02 / 09 — THE CHALLENGE

The Challenge

Friction kills. Seconds decide outcomes.

Across healthcare systems relying on mobile apps, emergency response begins with a friction chain that introduces avoidable, life-threatening delay. In high-stress situations — car accidents, sudden collapse, medical distress — this process is too slow.

03 / 09 — THE TWO NODES

The Two Nodes

Sensing separated from communication

The system splits into two dedicated components — each optimised for its role. This separation improves signal quality, reliability, and flexibility far beyond any single-device approach.

Chest Sensor Node — body-worn
Node 01
Chest Sensor Node
A lightweight chest attachment responsible for body-worn intelligence. Minimal processing, high signal accuracy, continuous low-power monitoring.
  • ECG monitoring (heart activity)
  • Heart rate tracking (PPG backup)
  • Fall detection
  • Sudden impact sensing
  • Immobility / collapse detection
  • Wireless transmission to hub
Emergency Hub — necklace / clip
Node 02
Emergency Hub
A compact MiFi-sized necklace or clip device acting as primary communicator, emergency trigger interface, and context transmitter.
  • One-press emergency button
  • GPS location tracking
  • Cellular communication (SIM / eSIM)
  • Microphone — priority audio capture
  • Camera — multi-snapshot transmission
  • Independent operation without chest node

04 / 09 — HOW IT WORKS

How It Works

Zero to response in under 2 seconds

The system operates in two phases: an instant trigger layer that fires in 0–2 seconds, and an escalation sequence that builds context in real-time as responders mobilise.

Instant trigger — 0 to 2 seconds A single button press immediately sends an emergency alert, transmits last known location, and delivers a basic health snapshot — via cellular data with SMS fallback for low-signal environments.

05 / 09 — KEY INNOVATIONS

Key Innovations

Five design decisions that change outcomes

Each innovation targets a specific failure mode in existing emergency systems — from interface friction to bandwidth constraints to network unreliability.

1
Frictionless Emergency Activation No app. No navigation. No unlock. One physical action triggers the full response chain. Designed for trembling hands, impaired cognition, and zero cognitive load under distress.
2
Distributed System Architecture Separating sensing (body) from communication (hub) improves signal quality, allows independent component failure without total system failure, and optimises each unit for its specific task.
3
Multi-Layer Communication Reliability LTE cellular as primary. SMS as fallback. Local buffering as final safety net. Designed specifically for real-world network conditions including regions with unstable connectivity.
4
Bandwidth-Aware Context Streaming Instead of heavy video streaming, the system uses multi-image burst transmission converted to video server-side. Visual context delivered without network overload.
5
Independent Operation Mode The hub functions fully without the chest node — acting as a standalone emergency communicator. Redundancy is built into the architecture, not bolted on.
